ssh连接github失败

在本地的 .ssh目录里存在若干账号的key，但是里面github key悄然失踪了，于是重新生成了一个。执行 ssh git@github.com 连接github，会提示Permission denied (publickey)。
是哪出错了？为了看到错误信息，执行 ssh -T -v git@github.com
这时可以看到，ssh尝试连接时只寻找了id_rsa这个key。这说明，ssh并没有把新生成的key添加到自己的key列表里。所以现在要做的就是把github的key添加进来。
先执行 

eval \`ssh-agent\`  （是～键上的那个\`）

再执行 

ssh-add ~/.ssh/key文件

添加成功， ssh-add -l 可以看到新加的rsa了。 再执行ssh git@github.com，成功连接到github！
